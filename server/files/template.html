ログ

ページごとに、何時、何を、誰がどのようにを変更したかをdiff形式で見られる。diffに対してコメントを付けられる　
logとコメントの組み合わせで、ブログのようなシステムが作れる。

サイトマップ

編集

サイトマップの編集・記事の編集が主な機能となり、　この2つにより、他の項目は自動生成される

専用言語

記事の作成で利用する言語。markdownのようなもの。Javascript/サーバ用のトリガの記述ができ、アップロード機能と合わせると、特殊な機能が必要な、トップページや通信用フォームさえも作れる

アップロード機能

HTML JS サーバ用機能　CSS 画像　など、木構造で管理される。 順序は並べた順につく。ファイルシステムが存在し、データベースにそれらへのポインタが含まれる。 全てのファイルに対しfilename@difとfilenameが作成される。　これにより、全てのファイルに対してdiffが発生する

記事の詳細データなどが入る

  • コメント page#comment@view
  • ページの編集 page#root@edit
  • 記事に関するデータの詳細　日時　階層　タグなど 履歴が要らなくなるかもしれない page#datas
