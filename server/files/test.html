URL

domain.com/path/to/page について、[path][#content][@eval-type] の形のURLをとり、PathはDBに登録されているパス名、contentはDBのpath項目にあるcontent名、eval-typeは評価の形式であるである。省略される場合がある 例として

  1. path/to/page の場合、メインフレーム内に、fileの形式に対応したデザインで、pageの内容を評価したテキストが配置される =content,typeが特定の場合　評価する
  2. path/to/page@file の場合、ダウンロードされる。ブラウザ上には主にプレテキストとして表示される場合がある　評価しない
  3. path/to/page#content の場合、contentで指定された要素を取り出して、pageと同様に処理する 評価する
  4. path/to/page#content@data の場合、contentで指定されたデータベースの要素を取り出して、@fileと同様の処理をする 評価する
  5. path/to/page#abstract-list@view の場合、子のなかのdataの概要を取り出して、pageと同様の処理をする 評価する

また、URLを用いたフォームなどの通信を禁止する。Ajaxを使うこと
DBはパス名と、実際のディレクトリへのパスを格納する。
